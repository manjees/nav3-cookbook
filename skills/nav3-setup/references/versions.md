## Metadata DSL Version Requirement

The type-safe metadata DSL (`metadata { put(NavMetadataKey, value) }`) and the
`NavMetadataKey` interface were introduced in `navigation3 1.1.0-alpha04` and are
part of the `1.1.0` stable release.

On `1.0.x` stable (no `NavMetadataKey`, no `metadata { }` DSL), entry metadata is
a plain `Map<String, Any>`, and transition overrides are expressed via the helper
functions on the `NavDisplay` object (all return `Map<String, Any>`):

- `NavDisplay.transitionSpec { ... }`
- `NavDisplay.popTransitionSpec { ... }`
- `NavDisplay.predictivePopTransitionSpec { ... }`

These helpers exist in both `1.0.x` and `1.1.x`, so they are the version-portable
way to set per-entry transitions.

## NavDisplay API differences — 1.0.x vs 1.1.0

Code generated by these skills targets the **1.1.0** API. Key differences from 1.0.x:

| Area | 1.0.x | 1.1.0 (current) |
|------|-------|-----------------|
| Scene strategy param | `sceneStrategy: SceneStrategy<T>` (singular) | `sceneStrategies: List<SceneStrategy<T>>` (plural list) |
| Scene decorator chain | unavailable | `sceneDecoratorStrategies: List<SceneDecoratorStrategy<T>>` |
| Shared transition | unavailable on `NavDisplay` | `sharedTransitionScope: SharedTransitionScope?` |
| Type-safe metadata | raw `Map<String, Any>` only | `NavMetadataKey<T>` + `metadata { put(key, v) }` DSL |
| Transition override keys | string constants (internal) | `NavDisplay.TransitionKey`, `PopTransitionKey`, `PredictivePopTransitionKey` — typed `NavMetadataKey` objects |

The old `NavDisplay(..., sceneStrategy = ..., ...)` overload still compiles on
1.1.0 but is `@Deprecated(level = HIDDEN)` and forwards to the plural overload.

## Known Compatibility Issues

- `compileSdk 35` or lower: some APIs are unsupported — upgrade to 36.
- Kotlin below 2.0: `@Serializable` processing differs; Kotlin 2.0+ is required
  for `rememberNavBackStack` to work reliably.
